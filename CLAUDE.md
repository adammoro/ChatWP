# CLAUDE.md

Guidance for AI assistants working in this repository.

## Overview

ChatWP is a small WordPress plugin that adds a sidebar widget which displays text
generated by OpenAI's (legacy) Completions API. Site admins enter an OpenAI API
key on a settings page, then add one or more "ChatWP Widget" instances via
Appearance → Widgets, each with its own prompt and generation parameters.

This is a tiny, single-file plugin — there is no build system, package manager,
or test suite. Treat it as plain PHP written against the WordPress plugin API.

## Repository structure

| `chatwp/chatwp.php`  | The entire plugin — widget class, settings page, OpenAI API call.    |

## Architecture (`chatwp/chatwp.php`)

The whole plugin lives in one file with three parts:

1. **`ChatWP_Widget`** (extends `WP_Widget`) — the widget itself.
   - `form()` renders the admin fields (title, prompt, model, temperature,
     max_tokens, frequency_penalty, presence_penalty) with sensible defaults.
   - `update()` sanitizes submitted fields with `strip_tags()` before saving.
   - `widget()` reads the saved instance settings, calls `chatwp_get_completion()`,
     and echoes the returned text inside `.chatwp-text`.
   - Registered via `register_chatwp_widget()` on the `widgets_init` hook.

2. **Settings page** — a single "API Key" field under Settings → ChatWP.
   - `chatwp_api_key_init()` registers the `chatwp_api_key` option (`admin_init`).
   - `chatwp_add_options_page()` adds the page under Settings (`admin_menu`).
   - `chatwp_options_page()` renders the form; the key is read elsewhere via
     `get_option('chatwp_api_key')`.

3. **`chatwp_get_completion()`** — builds and sends the OpenAI request.
   - POSTs to `https://api.openai.com/v1/completions` via `wp_remote_post`.
   - Returns the decoded JSON response object, or `false` if `wp_remote_post`
     returns a `WP_Error`.

## Development environment

There's no build step. To run/test changes, copy or symlink the `chatwp/`
folder into a local WordPress install's `wp-content/plugins/` directory and
activate "ChatWP" from the Plugins screen. There is no linter, formatter, or
automated test suite configured in this repo — verify changes manually inside
WordPress (activate the plugin, add a widget, check the settings page).

## Conventions

- Procedural functions + `add_action` hooks for settings/admin-menu wiring;
  OOP only for the widget itself (`WP_Widget` subclass), matching WordPress
  widget conventions.
- Sanitize widget input with `strip_tags()` in `update()`; escape output with
  `esc_attr()` in `form()`.
- Use `__()` / `_e()` for user-facing strings, with text domain `'chatwp'`.
- Keep the plugin to a single file unless it grows enough to justify splitting
  — there's currently no autoloading or includes mechanism set up.

## Distribution note

The README tells users they can install from either the raw `chatwp/` folder
or `ChatWP.zip`. If you modify `chatwp/chatwp.php`, remember `ChatWP.zip` is a
separate, manually-maintained copy — there is no script that regenerates it,
so update the zip too if you want the two install paths to stay in sync.

## Known quirks

In `chatwp_get_completion()`, `temperature`, `frequency_penalty`, and
`presence_penalty` are cast with `intval()` before being sent to OpenAI, even
though these are meant to be floats (e.g. `0.9`, `0.6`) and the widget form
defaults to fractional values. As written, `intval()` truncates them to `0`
or `1`. This is existing behavior, not something to silently "fix" as a side
effect of an unrelated change — flag it if you touch this function.
